■ 02 · Vấn đề 02 / 08

Problem statement

Không tồn tại print designer Vue
tự do thương mại hóa.

Yêu cầu bề mặt: "thiết kế lại vue-print-designer bằng Nuxt 4, design lại UI." Vấn đề thật nằm sâu hơn.

Repo tham chiếu là AGPL-3.0 + điều khoản thương mại riêng: bản open-source chỉ cho cá nhân phi thương mại, bắt buộc giữ logo và copyright. Muốn dùng doanh nghiệp hoặc bỏ logo — phải mua license từ printdot.cc.

Port code trực tiếp sang Nuxt 4 tạo ra derivative work, kế thừa toàn bộ ràng buộc đó. Hệ sinh thái Vue hiện không có print designer hiện đại, nhúng được, license thoáng — đó là khoảng trống sản phẩm này lấp.

Port code là mang xiềng xích AGPL về nhà. Clean-room là con đường duy nhất đến tự do license.

■ 05 · Khuyến nghị 05 / 08

Kiến trúc chốt

Library là sản phẩm.
App Nuxt là showcase sống.

pro-print-designer/            (pnpm workspace)
├── packages/editor/           # @pro-print/editor — Vue 3 lib (Vite lib mode)
│   ├── core/                  # schema template JSON, command pattern undo/redo
│   ├── components/            # canvas, elements, toolbar, properties panel
│   ├── render/                # measure → paginate → print iframe / ảnh / PDF
│   └── styles/                # Tailwind preset + design tokens
└── apps/web/                  # Nuxt 4 + Nuxt UI v3
    ├── landing                # SSR
    ├── template manager       # IndexedDB CRUD, import/export
    └── editor page            # ssr: false, host @pro-print/editor

State management

Pinia store module hóa — element / selection / history / page / guides. Cùng cách phân rã đã được kiểm chứng ở bản gốc, nhưng implementation hoàn toàn tự viết.

Vì sao Nuxt 4 hợp lý

Editor bản chất là SPA client-side — SSR vô nghĩa với canvas. Nuxt chỉ đáng giá vì user chọn app hoàn chỉnh: landing SSR, quản lý template, nền cho auth/sync tương lai. Trang editor chạy ssr: false để né toàn bộ rủi ro hydration.

■ 06 · Lộ trình 06 / 08

7 phase tuần tự

Full parity — nhưng
không bao giờ làm một cục.

  1. 01
    Nền tảng canvas
    Monorepo scaffold, page model, zoom/pan, rulers/guides, kéo-thả-resize-xoay, selection, undo/redo (command pattern).
  2. 02
    Element cơ bản + properties panel
    Text, image, line, rect, circle. Lưu IndexedDB, import/export JSON.
  3. 03
    Print pipeline v1
    Browser print, export ảnh/PDF, preview. Kiểm chứng mm↔px/DPI trên máy in thật ngay tại đây — không để đến cuối.
  4. 04
    Barcode + QR + page number
    jsbarcode, qrcode — hai dependency nhẹ đã kiểm chứng.
  5. 05
    Table + smart pagination
    Boss cuối — ~30% công sức toàn dự án. Tách riêng, tham khảo hành vi bản gốc (không phải code), chấp nhận nhiều iteration.
  6. 06
    Multi-label, phím tắt, dark mode, i18n
    VN/EN trước, mở rộng sau (bản gốc có 6 locale).
  7. 07
    Web component wrapper + publish npm
    Hoàn tất câu chuyện cross-framework — ngang hàng bản gốc về phân phối.

Ngoài scope: silent print / cloud print (cần desktop client — để mở qua WebSocket agent protocol tương lai), server template sync (vòng 2), Monaco scripting (quyết định ở phase 4+).
